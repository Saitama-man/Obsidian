[[Функциональное тестирование ]]  **"ЧТО делает система?"**
[[Нефункциональное тестирование  ]] **"КАК система это делает?"**

По степени формализации:
	**На основе тест-кейсов 
		(Scripted):** Тестирование по заранее подготовленным сценариям.  
	**Исследовательское (Exploratory):**
		 Тестировщик одновременно изучает продукт, проектирует и выполняет тесты.  
	**Свободное (Ad Hoc):**
		 Неформализованное тестирование, основанное на интуиции и опыте

По степени автоматизации:
	**Ручное:**
		 Тесты выполняются человеком вручную.  
	**Автоматизированное:**
		 Тесты выполняются с помощью скриптов и инструментов.

По запуску кода:
	**Статическое:**
		 Тестирование без запуска кода (проверка документации, ревью кода).  
	**Динамическое:**
		 Тестирование с запуском кода (проверка реального поведения приложения).

По принципам работы с приложением
	**Позитивное:**
		 Проверка, что система корректно работает при правильных действиях и данных.  
	**Негативное:**
		 Проверка, что система корректно обрабатывает ошибки при неверных действиях и данных.
